# Implementation Plan: Basic Goal Setting

**Branch**: `152-goal-setting` | **Date**: 2026-02-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/152-goal-setting/spec.md`

## Summary

Add a goal-setting system with four goal types (session frequency, practice time, item mastery, milestone) supporting full CRUD, computed progress from existing session data, a dedicated `/goals` page with create form, a 5th navigation tab, and an active goals summary card on the library home page. Goals use flat-column discriminated storage in SQLite, internally-tagged serde for API serialisation, and pure-function progress computation at view-build time.

## Technical Context

**Language/Version**: Rust stable (1.89.0), 2021 edition
**Primary Dependencies**: crux_core 0.17.0-rc2, serde 1, chrono 0.4, ulid 1, axum 0.8, libsql 0.9, leptos 0.8.x (CSR), Tailwind CSS v4
**Storage**: Turso (managed libsql/SQLite) via HTTP protocol — new `goals` table with flat columns
**Testing**: cargo test (unit in intrada-core, integration in intrada-api), wasm-bindgen-test 0.3
**Target Platform**: WASM (web shell) + Linux server (API)
**Project Type**: Web application (three-crate workspace: core, web, api)
**Performance Goals**: Goal CRUD operations and progress computation feel instant; no perceptible delay on a library of up to 10,000 items and 100 active goals
**Constraints**: Pure core (no I/O in intrada-core), WASM bundle size impact minimal, API response < 200ms p95
**Scale/Scope**: Single-user app, ~50 active goals max, 4 goal types, 2 new views, 1 summary card, 1 nav tab
